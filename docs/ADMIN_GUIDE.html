1. Первый день админа

  1. Зайти под своим логином в админку.
  2. Проверить вверху хедера: есть ли бейдж роли (owner / senior_admin / admin / specialist).
  3. Открыть Dashboard — убедиться, что видны выручка месяца, последние брони, аналитика.
  4. Открыть Bookings → Шахматка — проверить сегодняшний день.
  5. Открыть Finance → Баланс по счетам — увидеть текущее состояние кассы.
  6. Открыть CRM (кнопка в хедере) — проверить клиентов и долги.

2. Навигация и роли

Порядок вкладок: Dashboard → Bookings → Finance → Tasks → Clients → Specialists → Users → Settings.

РольЧто может
ownerВсё. Корректировка балансов, удаление любых транзакций, управление правами.
senior_adminВсё, кроме управления правами. Корректировка балансов, удаление любых транзакций.
adminБрони, касса, CRM. Удаление только сегодняшних транзакций.
specialistТолько свой CRM и расписание.

Owner/senior_admin получают авто-доступ к CRM без отдельной заявки.

3. Создание брони

  1. Bookings → Шахматка (или «Новая бронь» из дашборда).
  2. Выделить мышью нужный слот на шахматке (можно тащить по нескольким часам).
  3. В боковой панели выбрать клиента (если пользователь зарегистрирован) или ввести имя.
  4. Выбрать формат: Индивид. / Группа / Интервизия — от этого зависит тариф.
  5. Цена рассчитается автоматически (базовый тариф × длительность + пиковые часы − скидки − абонемент).
  6. Подтвердить → бронь появится на шахматке.

Админ может бронировать задним числом до 12 часов назад (например, забыли оформить раннюю бронь).

4. Перенос, продление, отмена

Перенос

В шахматке взять бронь мышью и перетащить на новое время. Откроется подтверждение.

Продление на +30 мин

В карточке брони — кнопка «+30 мин». Проверит конфликт со следующей бронью и добавит оплату по тарифу.

Отмена

  • В карточке брони → «Отменить».
  • В таймлайне будет видно время отмены, не дата самой брони.
  • Если бронь была «субарендой» — при отмене 50% возвращается автору, остальное → доход Unbox.

5. Горячие брони и одобрение

Горячая бронь = клиент бронит менее чем за N часов до начала.

  • Обычный клиент создаёт горячую бронь → она в очереди одобрения.
  • Админу приходит уведомление в колокольчик.
  • Админ открывает → Одобрить или Отклонить.
  • Админы сами создают горячие брони без одобрения.

6. Многочасовая бронь на нескольких клиентов

Если клиент забронировал 3 часа, но часть — для другого человека:

  1. Открыть бронь → «Разбить на слоты».
  2. Для каждого часа выбрать клиента (из CRM).
  3. Каждый час записывается как отдельная сессия CRM.
  4. На шахматке отображается разными цветами.

7. Повторяющиеся еженедельные брони

  1. При создании брони отметить «Повторять еженедельно».
  2. Указать количество недель (например, 8).
  3. Система создаст 8 броней — по одной на каждую неделю.
  4. Если неделя занята — эта неделя пропускается, остальные создаются.
  5. Отменить можно как всю серию, так и отдельную бронь.

8. Приход, расход в кассе

  1. Finance → Транзакции → «+ Транзакция».
  2. Выбрать тип: Приход / Расход.
  3. Выбрать счёт: Cash / TBC / BOG / Карта / Перевод / RU Банк.
  4. Выбрать категорию (категории фильтруются по типу транзакции).
  5. Указать сумму и валюту (GEL / USD / RUB).
  6. Филиал — Unbox Uni / Unbox One / Neo School.
  7. Можно привязать к клиенту CRM (для приходов).
  8. Сохранить.

Удаление

  • Админ — только сегодняшние.
  • Senior/owner — любые.

9. Пополнение баланса клиента

  1. Users → карточка клиента → «Пополнить баланс».
  2. Указать сумму, валюту, счёт и филиал.
  3. Транзакция автоматически появится в кассе с категорией «Пополнение баланса».
  4. Баланс клиента увеличится мгновенно.

10. Перевод между счетами

Нужно, если, например, деньги из Cash положили в TBC.

  1. Finance → Счета → «Перевод».
  2. Выбрать счёт-источник и счёт-получатель.
  3. Указать сумму.
  4. Создаются две связанные транзакции (расход + приход).

11. Открытие и закрытие смены

Открытие

В начале рабочего дня: Finance → «Открыть смену». Фиксируется стартовый остаток по каждому счёту и филиалу.

Закрытие смены

  1. Finance → «Закрыть смену».
  2. Модалка сама подставит филиал, в котором ты работал.
  3. Указать фактический остаток наличных (система сверяет с расчётным).
  4. Если есть расхождение — появляется поле «Комментарий».
  5. Закрытие создаёт отчёт смены.

Смены закрываются только по наличным. Безнал сверяется автоматически.

12. Корректировка баланса

Только для owner / senior_admin.

  1. Users → клиент → «Корректировать баланс» (или Finance → Балансы).
  2. Указать сумму (+/−), причину.
  3. Создаётся транзакция типа manual_correction.
  4. Видна в истории клиента.

13. Мульти-валютность

Клиент может быть в GEL / USD / RUB.

  • На дашборде выручка показывается в трёх колонках: 1200 ₾ · 50 $ · 3000 ₽ + GEL-эквивалент общей строкой.
  • В долгах тот же формат.
  • Пересчёт в GEL идёт по актуальному курсу из настроек.
  • Мин/Макс тариф на дашборде автоматически переводится в GEL для сравнения.

14. Клиенты: создание, объединение, удаление

Создание

  • CRM → Клиенты → «+ Клиент» или из страницы пользователей админки.
  • Поля: имя, контакты, базовый тариф, счёт по умолчанию, валюта, персональная скидка.
  • Поддержка инлайн-редактирования прямо в карточке.

Объединение (merge)

Если один клиент создан дважды.

  1. Открыть клиента → «Объединить с…».
  2. Выбрать второго клиента.
  3. Все сессии и платежи переносятся на основного, alias сохраняется (важно для Google Calendar).

Удаление

Кнопка «Удалить» в списке клиентов. С подтверждением. Все связанные сессии удаляются — осторожно.

15. Сессии

Добавить сессию

В карточке клиента → «+ Сессия». Указать дату, время, длительность, цену, счёт.

Оплата

  • Кнопка «Быстрая оплата» использует цену сессии. Если цена NULL — берётся базовый тариф клиента. Нулевые суммы пропускаются.
  • Можно выбрать счёт прямо в панели редактирования сессии.
  • «Отметить всё оплачено» — помечает все прошедшие сессии, будущие не трогает.

Завершение

Сессии автоматически завершаются по локальному времени (Батуми), когда заканчиваются.

Редактирование отменённых

Даже отменённые сессии можно редактировать и удалять.

Смена клиента в сессии

Кнопка «Сменить клиента» в окне редактирования (например, если записал на неправильного человека).

16. Долги

  • Долг = завершённая + неоплаченная сессия. Будущие и отменённые не считаются.
  • Кнопка «Неоплаченные сессии» на дашборде ведёт в раздел «Долги».
  • В списке клиентов долги отображаются в отдельной колонке.
  • Неактивные клиенты в суммарный долг не включаются.
  • Мульти-валютный вывод: долг 300 ₾ · 20 $.

17. Синхронизация с Google Calendar

Разовая синхронизация

  1. CRM → Сессии → «Sync» (модалка).
  2. Выбрать период (неделя / месяц / произвольный).
  3. Импортируются события → создаются сессии.

По клиенту

В карточке клиента → «Sync этого клиента» с выбором периода.

Автосоздание клиентов

Если в событии календаря фигурирует имя, которого нет в CRM — клиент создастся автоматически.

Слияние (merge) и sync

Если клиенты были объединены, их alias-коды (например, IVA01, IVA02) все учитываются при поиске событий.

Проверка дубликатов

Сессия не создастся повторно, если уже есть сессия того же клиента на ту же дату (даже если другой google_event_id).

18. Расписание специалиста и шахматка записи

Для специалиста

  1. CRM → Расписание.
  2. Задать недельный шаблон: для каждого дня — время начала, конца, филиал (или онлайн).
  3. Добавить исключения: конкретные даты (отпуск, доп. день).
  4. Переключатель «вкл/выкл» по дням.

Для клиента

  • Открывает страницу специалиста в каталоге.
  • Видит шахматку на 2 недели вперёд.
  • Свободные слоты кликабельны → модалка с формой (имя, телефон, email).
  • Занятые слоты — серые или помечены ×.
  • После записи — специалисту приходит Telegram-уведомление.

Отмена

Клиент или админ отменяет запись → уведомление уходит обеим сторонам.

19. Пользователи и роли

Users → список пользователей.

  • «+ Новый клиент» — прямо из админки без самостоятельной регистрации.
  • Роль выбирается из выпадающего: user / admin / senior_admin / owner / specialist.
  • У роли — дефолтные права (можно переопределить отдельно).
  • Бейдж «На проверке» скрыт для админов — вместо него показывается бейдж роли.
  • Смена пароля — либо админ из карточки, либо пользователь сам в профиле.

20. Каталог специалистов

Admin → Specialists. Каждая анкета:

  • Фото, имя, tagline (до 150 символов), био.
  • Категория: психология / психиатрия / наркология / коучинг / образование.
  • Специализации (теги), форматы (индивид./групп./интервизия).
  • Базовый тариф.
  • Порядок сортировки (меньше = выше в каталоге).
  • Привязка к пользователю — чтобы специалист мог сам править расписание.
  • Verified-флаг (если выключен — не показывается в публичном каталоге).
  • Тогглы «Показать/Скрыть» и «Удалить».

Карточка в каталоге — соотношение 3:4.

21. Абонементы, цены, скидки

Страница /subscriptions — 5 планов + стандартные цены.

Скидки применяются по правилу MAX — не складываются, выигрывает самая крупная:

  • Пиковые часы (надбавка).
  • Горячая бронь (надбавка).
  • Последовательные часы (скидка).
  • Еженедельная прогрессивная (чем больше броней в неделю — тем больше скидка).
  • Персональная скидка клиента.
  • Абонемент.

Welcome-бонус: 1 бесплатный час автоматически выдаётся новому пользователю при регистрации.

Инфографика со всеми правилами на странице абонементов — ссылка в футере.

22. Дашборд админа

  • Блоки можно перетаскивать (drag & drop), менять размер.
  • Чекбоксы в настройках — какие блоки показывать.
  • «iPhone-стиль» — долгое нажатие → блоки «колышутся», можно переставлять.
  • Выручка месяца считается на бэкенде (не ограничена 50 транзакциями).
  • Источник выручки — касса, не брони (избегаем двойного счёта).

23. Telegram-бот

Команды

  • /start — привязать Telegram к аккаунту.
  • /book — мастер бронирования кабинета (работает как сайт).
  • /balance — баланс.
  • /mybookings — мои брони.

Мастер /book

  1. Филиал
  2. Формат
  3. Дата
  4. Кабинет
  5. Время
  6. Длительность
  7. Подтверждение (с полной разбивкой цены)
  8. Бронь

Цена считается по тем же правилам, что и на сайте (PricingService). После подтверждения — бронь сразу отражается в системе и уходит в Google Calendar.

Уведомления специалистам

  • Новая запись от клиента → сообщение в Telegram (имя, контакты, время, заметки).
  • Отмена записи → уведомления обеим сторонам (специалист + клиент, если у него есть TG).

24. Уведомления

Колокольчик в хедере админки. Показывает:

  • Новые горячие брони на одобрение.
  • Новые заявки в CRM (со ссылкой на вкладку специалистов).
  • События таймлайна.

Клик по уведомлению → переход к соответствующей записи.

25. Типичные ошибки

СимптомРешение
Транзакции не отображаютсяПроверить фильтр по дате в Finance. По умолчанию — текущий месяц.
Долг у клиента не меняетсяОбновить страницу. Платёж на 0 ₾ не учитывается.
GCal не синкаетПроверить, что credentials.json на проде. В логах должно быть Authenticated via file: credentials.json.
Клиент появился дваждыИспользовать merge в карточке клиента.
Бронь не создаётся — «слот занят»Обновить шахматку — возможно, кто-то забронил.
LTV показывает 0Это уже пофикшено (использует прямой SQL). Если видишь — перезапустить бэк.
Горячая бронь «зависла»Проверить уведомления — она в очереди одобрения.
Смена не закрываетсяПроверить филиал в модалке. Возможно, открыта в другом филиале.
