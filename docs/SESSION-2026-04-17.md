## Что прилетело в прод (по блокам)

### 1. Косметика и ликвидация "черновиков"
- Убраны `MMXXVI`, `Черновик`, `Grid House · Направление A · Эксперимент`
  со всех клиентских страниц.
- Welcome Gate: `Я клиент` / `Я специалист` (с подписями).
- Реальные контакты в футере: 2 адреса, `+995 599 324 668`,
  `unbox.psy@gmail.com`, TG / IG / FB.
- KnowledgeBase: тарифы обновлены (Пробный 70₾, Тёплый старт 180₾,
  Регулярный 350₾, Профи+ 650₾, Групповой 450₾), «overstay» > 5 мин.
- Дубликат кнопки «Мои бронирования» с дашборда убран; «Новое
  бронирование» ведёт в `/booking`.

### 2. Hot booking: убрали скидку, оставили админ-одобрение
- `pricing.py` — `hot_booking.percent = 0`, `hot_percent` удалён из
  расчёта best-discount.
- `bookings/routes.py` — approval gate переведён с `quote.applied_rule`
  на прямой `utcnow() <= start_dt < utcnow() + 12h` (tz-aware UTC).

### 3. Telegram-бот + Email-уведомления
- `TelegramService` — подтверждения броней, уведомления специалистов,
  уведомления waitlist; graceful False если чат не подключён.
- `EmailService` — SMTP HTML + plain-text, TLS/SSL, отключаемо флагом
  `EMAILS_ENABLED`.
- `POST /telegram/link-token` — deep-link привязка
  (`https://t.me/<BOT>?start=<token>`), TTL 10 мин.
- `POST /telegram/webhook` — валидирует `X-Telegram-Bot-Api-Secret-Token`;
  обрабатывает `/start`, `/book`, `/bookings`, `/balance`, `/locations`,
  `/specialists`, `/help`.
- **Booking-flow в боте:** location → format → date → resource → time
  → duration → confirm через inline-клавиатуры.
- Frontend: кнопка «Подключить Telegram» в профиле (classic + GH),
  polling `currentUser.telegramId`.
- Интеграция: после `booking.status=confirmed` TG + email уходят
  fire-and-forget.

### 4. Intervision — третий формат
- `Format` type: `'individual' | 'group' | 'intervision'`
- Базовая ставка: 30₾/час для ROOM; для CAP — 10₾.
- Пропагировано: `FormatDateStep` (3-я карточка), `Summary`,
  `LocationDetailsPage`, `AdminAnalyticsCharts`, `UserBookingsTab`,
  `init_data.INITIAL_RESOURCES`.

### 5. Specialist schedule overrides
- CRM «Расписание»: weekly шаблон + override'ы на конкретные даты
  (отпуск, допдни, спецчасы).
- Backend: недельные слоты + `specific_date` override с приоритетом.

### 6. Privacy, security, hardening
- `BookingPublicRead` DTO без email/UUID для `/bookings/public`.
- Drag-guard: админ не перетаскивает чужие брони на своём чесс-борде.
- Auth: 24-часовая проверка свежести Telegram `auth_date`; токен из
  fallback-редиректа убран.
- Upload endpoints: обе точки требуют auth.
- **Advisory lock** на создании брони: `pg_advisory_xact_lock` по
  SHA-256(resource:date) — реально защищает от phantom-row race
  (SELECT FOR UPDATE на пустых слотах не помогает).
- Guest checkout email-regex (до этого `.trim()` пропускал `abc`).
- Route aliases: `/profile → /dashboard/profile`, `/bookings →
  /dashboard/bookings` (TG-бот и email шлют короткие пути).

### 7. UX квик-wins
- `/dashboard/bookings` для админа: только **свои** брони (раньше все).
- `listForReRent`: был локальный Zustand-мьют, теперь вызывает
  `bookingsApi.toggleReRent` — статус «переаренды» сохраняется на сервере.
- `PromptModal` — замена native `window.prompt()`.
- `DashboardLayout`: непрозрачный фон сайдбара + shadow на мобиле.
- `AdminCrm` revenue-фильтр: только `incomeTypes` (раньше double-counted).
- `Waitlist page`: `alert/confirm` → `toast + ConfirmationModal`.
- `CrmSessions`: KPI на 2 строки на мобиле.

### 8. Уведомления в Grid House
- `NotificationBell` получил `variant: 'light' | 'dark'`.
- Смонтирован в Grid House AdminLayout (вариант light — тёмная иконка
  на светлом фоне шапки).

### 9. QuickActionsFab — плавающее меню быстрых действий
- Новый компонент: круглая `⚡`-кнопка правый-нижний угол,
  раскрывается в меню.
- **CRM:** Добавить клиента → Запланировать сессию → Забронировать
  кабинет → **Открыть Google Calendar** (личный календарь специалиста
  если `calendarId` задан, иначе общий GCal).
- **Dashboard:** Забронировать → Мои брони → Найти специалиста.
- ESC / outside-click закрывают.

### 10. Waitlist → TG-уведомление когда слот освободился
- `TelegramService.send_slot_available` — HTML-сообщение с датой,
  окном, прямой ссылкой на `/booking`.
- `services/waitlist_notify.notify_waitlist_for_freed_slot`:
  - Ищет active записи с overlap окна по resource.
  - Каждой: TG-push (если подключён) + in-app Notification (всегда).
  - Помечает `fulfilled` — анти-спам.
- Интеграция в `cancel_booking`, `reschedule_booking` (старый слот
  освободился), `toggle_re_rent` (false → true).
- WaitlistModal: плашка статуса TG (зелёная «подключён» / янтарная
  нудж).

### 11. Cashbox → баланс клиента (reversible)
- `CashboxTransaction.credited_user_id` — маркер что транзакция
  пополнила баланс.
- `CashboxTransactionCreate.credit_user_balance: bool`.
- `create_transaction`: при `type=income + client_id + флаг=true` →
  резолвит клиента (UUID или email), `User.balance += amount`,
  записывает `credited_user_id`.
- `delete_transaction`: откат кредита.
- `update_transaction`: delta-коррекция / полный откат при смене type;
  переназначить клиента запрещено.
- UI: чекбокс в `AddCashboxTransactionModal` (on по умолчанию).

### 12. Admin change-email (cascade across soft FKs)
- `POST /users/{id}/change-email` (senior_admin / owner only).
- Обновляет email в 6 таблицах в одной транзакции:
  `booking.user_id`, `booking.cancelled_by`, `waitlist.user_id`,
  `cashbox_transactions.client_id`, `user.responsible_admin_id`,
  `user.attracted_by_admin_id`.
- Аудит-запись в `comment_history` с old/new + cascade rowcounts.
- UI: кнопка в панели «Безопасность» UserDetails.
- UUID-ссылки (`user_uuid`, `credited_user_id`, `recipient_id`)
  намеренно не трогаем — стабильны.

### 13. Shift close: двухэтапность + корректный discrepancy
- **Step 1: PreCloseShiftChecklist** — 5 пунктов (брони /
  транзакции / пересчёт / кабинеты / запереть). Прогресс-бар,
  «Дальше» disabled пока не все ☑.
- **Step 2: EndShiftModal** (existing).
- **Discrepancy fix:** `ShiftReport.branch` column + миграция +
  бэкфилл из legacy `[Branch]`-префикса в notes. `end_shift` теперь
  берёт `last_shift` с фильтром по scope (branch/global). Бэкфилл на
  проде корректно распределил 21 исторический отчёт: 10 global +
  5 Unbox One + 6 Unbox Uni.

### 14. Google Calendar 2-way sync (#15 / #32 / #38)
- **Push** (уже работал): при confirm booking → `gcal_service.create_event()`
  через Service Account.
- **Pull** (новое): `GoogleCalendarService.list_events(...)` реально
  вызывает Google API, фильтрует отменённые / all-day / наши собственные.
- Новый endpoint `GET /bookings/external-events`.
- Фронт: `src/services/googleCalendarMock.ts` больше не мок, кэш 5 мин.
- `TimelineStep` + `ChessboardStep` тянут окно ±1 день (TZ edge).
- **На проде endpoint вернул 8 реальных событий** для тестового кабинета
  с корректными датами (проверено).

### 15. Reschedule wizard (#27)
- Корневой баг: `startEditing` делал `selectedSlots: []` →
  `ConfirmationStep` строил корзину только из них → «Корзина пуста».
- Фикс: `startEditing` генерирует 30-мин слоты из `(resource_id,
  start_time, duration)` исходной брони. Пользователь видит свою бронь
  предвыбранной в шахматке.

### 16. PhoneInput — Georgia mobile mask (#39)
- Новый компонент `<PhoneInput>` без зависимостей; маска

- `formatGeorgiaPhone(raw)` — чистая функция.
- Встроен в 7 мест: гостевой checkout, LoginPage (classic + GH
  через GHField), ProfilePage (classic + GH), admin/Users create,

### Техдолг сервера устранён
До сессии на `/var/www/unbox-beta/backend` висело **28 файлов /
1303 незакоммиченных строки**. Выполнено в середине сессии:
1. `rsync` серверного бэкенда → `/tmp/server-drift/`
2. `diff` против локали — разница минимальна (только хирургические
   патчи hot-booking, которые я сам туда клал)
3. `git reset --hard origin/beta` на сервере (bak секретов в
   `/root/backup-*`)
4. Все правки оформил в коммиты локально, запушил, подтянул на сервер
   обычным `git pull`.

Теперь каждый деплой — чистый `git pull + systemctl restart`.

### Деплой-цепочка
```
локально → git push origin feature/grid-house:beta
         → rsync dist/ root@138.68.111.248:/var/www/unbox/dist/
сервер   → cd /var/www/unbox-beta && git pull origin beta
         → systemctl restart unbox-api
         → (если порт занят)  lsof -i :8000 -t | xargs -r kill -9
```
