## Что поехало в прод (по смысловым блокам)

### 1. Косметика и ликвидация "черновиков"
- Убраны `MMXXVI`, `Черновик`, `Grid House · Направление A · Эксперимент`
  со всех клиентских страниц.
- Welcome Gate: `Я клиент` / `Я специалист` (с подписями).
- Реальные контакты в футере: 2 адреса, `+995 599 324 668`,
  `unbox.psy@gmail.com`, TG/IG/FB.
- KnowledgeBase: тарифы обновлены (Пробный 70₾, Тёплый старт 180₾,
  Регулярный 350₾, Профи+ 650₾, Групповой 450₾), «overstay» >5 мин.
- Дубликат кнопки «Мои бронирования» с дашборда убран; «Новое
  бронирование» теперь ведёт в `/booking` (раньше в `/dashboard/bookings`).

### 2. Hot booking: убрали скидку, оставили админ-одобрение
- `pricing.py` — `hot_booking.percent = 0`, `hot_percent` удалён из
  расчёта best-discount.
- `bookings/routes.py` — approval gate переведён с `quote.applied_rule`
  на прямой `utcnow() <= start_dt < utcnow() + 12h` (tz-aware).

### 3. Telegram-бот + Email-уведомления
- **Services:**
  - `TelegramService` — подтверждения броней, уведомления специалистов,
    уведомления waitlist, graceful degradation.
  - `EmailService` — SMTP HTML + plain-text, TLS/SSL, отключаемо флагом
    `EMAILS_ENABLED`.
- **Bot API:**
  - `POST /telegram/link-token` — deep-link привязка
    (`https://t.me/<BOT>?start=<token>`), TTL 10 мин.
  - `POST /telegram/webhook` — валидирует
    `X-Telegram-Bot-Api-Secret-Token`; обрабатывает `/start`, `/book`,
    `/bookings`, `/balance`, `/locations`, `/specialists`, `/help`.
- **Booking-flow в боте:** location → format → date → resource → time
  → duration → confirm через inline-клавиатуры.
- **Frontend:** кнопка «Подключить Telegram» в профиле
  (classic + Grid House), polling `currentUser.telegramId` после открытия
  deep-link.
- **Интеграция:** после `booking.status=confirmed` в `create_booking`
  уходит TG + email (fire-and-forget).

### 4. Intervision — третий формат
- `Format` type: `'individual' | 'group' | 'intervision'`
- Базовая ставка: 30₾/час (ROOM); индивидуальные кабинеты по 10₾.
- Пропагировано: `FormatDateStep` (3-я карточка), `Summary`,
  `LocationDetailsPage`, `AdminAnalyticsCharts`, `UserBookingsTab`,
  `init_data.INITIAL_RESOURCES`.

### 5. Specialist schedule overrides
- CRM `Расписание`: weekly шаблон + override'ы на конкретные даты
  (отпуск, допдни, другие часы).
- `specialist_schedule.py` backend: недельные слоты + `specific_date`
  override с приоритетом.
- GridHouse и classic UI.

### 6. Privacy, security, hardening
- `Booking` приватность: `BookingPublicRead` DTO без email/UUID для
  `/bookings/public`.
- Drag-guard: админ не перетаскивает чужие брони в своём чесс-борде.
- Auth: 24-часовая проверка свежести Telegram `auth_date`; токен из
  fallback-редиректа убран.
- Uploads: обе точки требуют auth.
- Advisory lock на создании брони: `pg_advisory_xact_lock` по
  SHA-256(resource:date) — реально защищает от phantom-row race
  (в отличие от `SELECT FOR UPDATE` на пустых слотах).
- Guest checkout email-regex (до этого `.trim()` пропускал `abc`).
- Route aliases: `/profile → /dashboard/profile`, `/bookings →
  /dashboard/bookings` (TG бот и email шлют короткие пути).

### 7. UX квик-wins
- `/dashboard/bookings` для админа: показывает только **свои** брони
  (раньше все), чтобы клиентские брони админа не попадали в «мой список»
  и не создавали ложные коллизии на чесс-борде.
- `listForReRent`: был только локальный Zustand-мьют, теперь вызывает
  `bookingsApi.toggleReRent` — статус «на переаренде» больше не сбрасывается
  при переключении видов.
- `PromptModal` — замена native `window.prompt()`, используется в Waitlist.
- `DashboardLayout`: непрозрачный фон сайдбара на мобильных + shadow.
- `AdminCrm` revenue-фильтр: считает только `incomeTypes` (раньше
  double-counted).
- `Waitlist` page: нативные `alert/confirm` заменены на `toast` +
  `ConfirmationModal`.
- `CrmSessions`: KPI-карточки на мобильном переносятся на 2 строки.

### 8. Уведомления в новом дизайне (Grid House)
- `NotificationBell` получил `variant: 'light' | 'dark'`.
- Вмонтирован в Grid House AdminLayout (variant=light — тёмная иконка
  на светлом фоне шапки).

### 9. QuickActionsFab — закреплённое меню быстрых действий
- Новый компонент в правом нижнем углу, круглая кнопка ⚡.
- **CRM:** Добавить клиента → Запланировать сессию → Забронировать кабинет
  → **Открыть Google Calendar** (личный календарь специалиста если
  `calendarId` задан, иначе общий GCal).
- **Клиентский Dashboard:** Забронировать → Мои брони → Найти специалиста.
- ESC / outside-click закрывают.

### 10. Waitlist → уведомление когда слот освободился
- `TelegramService.send_slot_available` — HTML-сообщение с ресурсом,
  датой, окном, прямой ссылкой на бронирование.
- `services/waitlist_notify.notify_waitlist_for_freed_slot`:
  - Ищет активные записи по resource+date с пересекающимся окном.
  - Для каждой: TG-push (если подключён) + in-app Notification (всегда).
  - Помечает запись `fulfilled` — больше не спамит.
- **Интеграция в** `cancel_booking` (слот освободился), `reschedule_booking`
  (старый слот освободился) и `toggle_re_rent` (false → true transition).
- WaitlistModal: статус TG-плашка (зелёная «подключён» / янтарная с
  нуджем подключить).

### 11. Cashbox → баланс клиента (reversible)
- `CashboxTransaction.credited_user_id` — помечает транзакции, которые
  пополнили чей-то баланс.
- `CashboxTransactionCreate.credit_user_balance: bool` — опциональный
  флаг; при true + `type=income` + валидный `client_id` кредитует
  `User.balance += amount`.
- `delete_transaction`: откат кредита.
- `update_transaction`: delta-коррекция баланса при изменении суммы,
  полный откат при смене type→expense; переназначение клиента на
  кредитной транзакции запрещено (удалить + создать).
- UI в `AddCashboxTransactionModal`: чекбокс «Зачислить на баланс
  клиента» (по-умолчанию on). Когда off — зелёный текст меняется на
  янтарный «прочий приход».

### 12. Admin change-email (cascade across soft FKs)
- `POST /users/{id}/change-email` (senior_admin/owner).
- В одной транзакции обновляет email в 6 таблицах:
  `booking.user_id`, `booking.cancelled_by`, `waitlist.user_id`,
  `cashbox_transactions.client_id`, `user.responsible_admin_id`,
  `user.attracted_by_admin_id`.
- Аудит-запись в `comment_history` с old/new + cascade rowcounts.
- UI: кнопка в панели «Безопасность» UserDetails.
- UUID-ссылки (`user_uuid`, `credited_user_id`, `recipient_id`)
  намеренно не трогаем — они стабильны.

### 13. Shift close: двухэтапность + корректный discrepancy
- **Step 1: PreCloseShiftChecklist** — вечерний чек-лист из 5 пунктов
  (брони / транзакции / пересчёт наличных / кабинеты / запереть).
  Прогресс-бар + кнопка «Дальше — сверка кассы» disabled пока не все ☑.
- **Step 2: EndShiftModal** (существующая).
- **Discrepancy fix:** `ShiftReport.branch` column + миграция +
  бэкфилл из legacy `[Branch]`-префикса в notes. `end_shift` теперь
  берёт `last_shift` с фильтром по scope (branch/global), что убирает
  "постоянные расхождения". На проде бэкфилл корректно распределил
  21 исторический отчёт: 10 global + 5 Unbox One + 6 Unbox Uni.

1303 строки незакоммиченных правок** — мина под любой `git pull`. В
середине сессии сделал:
1. rsync серверного бэкенда → `/tmp/server-drift/`
2. diff против локали → разница минимальна (только хирургические
   патчи hot-booking, которые я сам туда клал)
3. `git reset --hard origin/beta` на сервере (bak секретов сначала
   в `/root/backup-*`)
4. Все правки оформил в коммиты из локали, запушил, подтянул на сервер
   обычным `git pull`

После этого каждый деплой — чистый `git pull + systemctl restart`.

### Деплой-цепочка
```
локально → git push origin feature/grid-house:beta
         → rsync dist/ root@138.68.111.248:/var/www/unbox/dist/
сервер   → cd /var/www/unbox-beta && git pull origin beta
         → systemctl restart unbox-api
         → (если порт занят)  lsof -i :8000 -t | xargs -r kill -9
```
